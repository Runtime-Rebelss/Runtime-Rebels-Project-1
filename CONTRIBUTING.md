Thanks for your interest in contributing to Runtime Rebels Project.

How to contribute

1. Fork the repository and create a branch for your change:
   git checkout -b feature/your-feature

2. Run the project and tests locally (see README for setup).

3. Open a pull request describing your change and link related issues.

Coding standards

- Keep commits small and focused.
- Write clear commit messages.
- Follow existing code style. For Java, use the project's formatter if provided.

Reporting bugs

- Open an issue with steps to reproduce, expected behavior, and environment.

Thank you for contributing!
